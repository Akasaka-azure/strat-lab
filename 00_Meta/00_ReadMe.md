

## 1. フォルダ階層と役割

|**フォルダ名**|**役割**|**Cursor（AI）への活用**|
|---|---|---|
|**`00_Meta`**|テンプレートや画像・添付ファイルの保管庫。|自動生成の型を定義する。|
|**`01_Inbox`**|**【入口】** 迷ったらここ。仕分け前の未整理メモ、一時的な保存。|AIに「ここを整理して」と指示する対象。|
|**`10_Daily`**|**【時間軸】** 日報、作業ログ、その日のタスク、電話メモ。|「最近の進捗」をAIに把握させるソース。|
|**`20_Fleeting`**|**【断片】** 思考の走り書き。アイデアの種。未完成の草案。|アイデアの膨らませを依頼する下書き。|
|**`30_Literature`**|**【外部】** 読書メモ、Webクリップ、技術記事の引用。|技術的な裏付けやソースとして参照させる。|
|**`40_Permanent`**|**【結晶】** 自分の言葉で定義した知識。strat-lab独自の開発規約。|**AIが最も重視すべき「正解」の基準。**|
|**`50_Projects`**|**【実務】** **Pythonコード本体**と、プロジェクト固有の仕様書。|コードと設計思想を紐付けて開発を行う。|
|**`60_Business`**|**【運営】** 経営、経理、顧客管理、開業戦略。|事業継続のための判断基準として参照。|
|**`70_Frameworks`**|**【基盤】** 自分のOS（資質、兵法、人的資本規格など）。|**あなたの性格・戦略に合わせた提案をさせる。**|
|**`80_Maps`**|**【構造】** 索引、全体を俯瞰する構造ノート、ダッシュボード。|AIに「全体像を把握」させるためのコンパス。|
## 2. 運用フロー（情報の昇華）

1. **収集 (`01_Inbox` / `10_Daily`)**: 全ての刺激をまずはここへ。
    
2. **整理 (`20_Fleeting` / `30_Literature`)**: 外部情報や断片的な思考を、特定のテーマに分類。
    
3. **結晶化 (`40_Permanent`)**: 自分の経験とフレームワーク（孫子等）を掛け合わせ、独自の知恵にする。
    
4. **実行 (`50_Projects`)**: 結晶化した知恵を使い、Pythonコードとして具現化する。
